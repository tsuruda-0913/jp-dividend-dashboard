# 高配当株ダッシュボード

日本の高配当株を「**買い時か？**」「**配当は安全か？**」の2つの視点で分析するダッシュボードです。
銘柄リストは Google スプレッドシートから自動取得し、株価・配当データは yfinance（Yahoo Finance）から取得します。
米国は個別株ではなく **高配当・増配ETF（HDV / SPYD / VYM / SCHD / VIG / VTI）** の比較分析ページで対応します。

## 画面構成

### 📈 トップ（銘柄一覧）
- **KPI 3枚** — 平均配当利回り / 利回りが過去レンジ上位の銘柄数（割安候補の目安） / 本日の最大下落
- **銘柄一覧テーブル** — 下落が大きい順。行クリックで個別分析ページへ
  - **前日比** — 下落の深刻度で色分け（5〜10% 黄 / 11〜15% 橙 / 16〜20% 濃橙 / 20%超 赤）
  - **利回り位置（3年）** — 現在の配当利回りが過去3年レンジのどの位置にあるかのバー表示。100%に近いほど過去比で高利回り＝割安寄り
  - **52週高値比** — 高値からの下落率。買い場探しの参考
  - **配当増減（TTM）** — 直近12か月の配当を前の12か月と比較。マイナスは減配の可能性（参考値）
  - 「財務指標も表示」トグルで ROE・自己資本比率・流動比率なども確認可能
- **下落TOP10 / 配当利回りTOP10** グラフ、**ニュース**タブ

### 🔎 個別銘柄分析（銘柄カルテ）
1ページ縦積みで、上から順に見れば買い時判断が完結する構成です。
1. **💹 買い時サマリー** — 株価チャート（20日MA±1σ/±2σ ボリンジャーバンド＋75/200日MA、6か月〜5年切替）と、**配当利回りレンジ**（過去5年のレンジ内で現在の利回りがどの位置か）を横並びで突合。**利回り→株価の逆算テーブル**で指値の目安も表示
2. **💰 配当の質** — 年間配当の推移（**減配年は赤で明示**）・EPSと配当性向・**FCF配当性向**（配当がフリーCFに収まっているか）・増配率（3年/5年）と5年後の想定YOC
3. **🏥 財務詳細** — 業績・財務の推移（折りたたみ）

> 利回りが過去レンジ上位（割安寄り）でも、減配リスクが見える場合は「利回りの罠」の可能性があります。①と②はセットで確認する設計です。

### 🇺🇸 米国ETF分析
HDV / SPYD / VYM / SCHD / VIG / VTI の6本を、セクター構成・上位銘柄の重複・分配金推移で比較。個別ETFはボリンジャーバンド付き株価チャートで買い時の参考位置（σ）も表示します。

## 使い方（Mac）

ターミナルでこのフォルダに移動して、次を実行するだけです。

```bash
./run.sh
```

初回は仮想環境の作成とパッケージのインストールが自動で走ります（数分かかります）。
起動するとブラウザで `http://localhost:8501` が開きます。停止は `Ctrl+C`。

> `permission denied` と出る場合は、最初に一度だけ `chmod +x run.sh` を実行してください。

### 手動で動かす場合

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
streamlit run app.py
```

## スマホから見る（Streamlit Community Cloud で無料公開）

GitHub にコードを上げると、Streamlit 社の無料ホスティングで常時公開でき、スマホのブラウザから見られます。

1. **GitHub にリポジトリを作成して push**
2. [share.streamlit.io](https://share.streamlit.io) にGitHubアカウントでログイン
3. 「New app」→ 対象リポジトリ／ブランチ（main）／Main file path に `app.py` を指定 → **Deploy**
4. 発行された `https://〜.streamlit.app` をスマホで開く

> Google スプレッドシートは「リンクを知っている全員が閲覧可」のままにしてください（クラウドからも読み込むため）。

## 監視銘柄の変更（Google スプレッドシート連携）

- 対象シート: [スプレッドシートを開く](https://docs.google.com/spreadsheets/d/1rI-X6hIi_FhThMoYGYEDWVkvJHqVMCZQED8_lDweUgw/edit)
- 必要な列: 「コード」「銘柄名」。任意で「配当利回り(%)」「市場」「セクター（または業種）」「ROE」「自己資本比率」など
  - 「セクター」列を追加すると一覧テーブルにセクターが表示されます（API取得はしないため高速なまま）
- シートは **「リンクを知っている全員が閲覧可」** にしておく必要があります
- 別のシートを使う場合は [stocks.py](stocks.py) の `SHEET_ID` を差し替えてください
- シートが読めない場合はフォールバック銘柄（3件）で起動し、画面上部に警告が出ます

## 補足

- 株価・配当は `yfinance`（Yahoo Finance）から取得します。**15〜20分ほど遅延**することがあります
- **配当履歴には中間配当の欠落等の誤差があり得ます**。減配判定・利回りレンジは参考値です（減配なし年数は履歴7年以上のときのみ表示）
- 名古屋証券取引所の専属銘柄など Yahoo Finance にデータが無い銘柄は「—」表示になります
- 「今すぐ更新」は一覧とニュースのキャッシュのみクリアします（個別銘柄・ETFのキャッシュは1時間で自動更新）
- 表示は参考情報であり、投資助言ではありません。投資判断は証券会社・運用会社の公式情報をご確認ください
